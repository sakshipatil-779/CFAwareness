# Judge Evidence: EcoQuest — Carbon Footprint Awareness Platform

This document maps each hackathon evaluation criterion to concrete implementation
evidence in the codebase, making it easy for judges to verify project quality.

---

## 1. Problem Statement Alignment

**Criterion**: Does the project directly address the stated problem?

**Evidence**:
- The problem — "Individuals lack concrete, immediate visual understanding of how
  daily micro-decisions (diet, commuting, energy) translate to global carbon
  footprints, leading to educational fatigue and inaction" — is addressed through
  a gamified simulation with real-time AI-generated visual feedback.
- Each user decision (morning commute mode, lunch choice, home energy setting)
  directly maps to a calculated carbon score, displayed on `ScoreBar.tsx`.
- A custom Vertex AI Veo 3.1 video is synthesized per user session, visualising
  the *exact* environmental future shaped by their choices.
- Three actionable reduction tips are generated by Gemini Flash after each
  scenario, providing immediate, real-world guidance.

**Files**:
- [`frontend/src/data/scenarios.ts`](../frontend/src/data/scenarios.ts) — all scenario decision trees
- [`frontend/src/components/game/ScoreBar.tsx`](../frontend/src/components/game/ScoreBar.tsx) — live carbon score display
- [`backend/src/services/videoService.ts`](../backend/src/services/videoService.ts) — Veo video synthesis
- [`docs/PROBLEM_ANALYSIS.md`](./PROBLEM_ANALYSIS.md) — full problem breakdown

## 3. Code Quality

**Criterion**: Is the codebase well-structured, maintainable, and follows best practices?

**Evidence**:

### TypeScript Strict Mode
Both `backend/tsconfig.json` and `frontend/tsconfig.app.json` enforce:
```json
{
  "strict": true,
  "noUnusedLocals": true,
  "noUnusedParameters": true,
  "noImplicitReturns": true,
  "noFallthroughCasesInSwitch": true
}
```

### ESLint + Prettier
- Backend: [`backend/.eslintrc.json`](../backend/.eslintrc.json) — `@typescript-eslint/recommended-requiring-type-checking` ruleset
- Frontend: [`frontend/.eslintrc.json`](../frontend/.eslintrc.json) — React + accessibility + TypeScript rules
- Formatting: [`.prettierrc`](../.prettierrc) — enforced across all 23 source files in CI

### CI/CD Pipeline
- GitHub Actions: [`.github/workflows/ci.yml`](../.github/workflows/ci.yml)
- Steps: `npm ci` → `prettier --check` → `eslint .` → `tsc --noEmit` → Docker build → Firebase deploy

### Architecture
- **Decoupled monorepo**: `backend/` (Express + TypeScript) and `frontend/` (Vite + React 19)
- **In-memory LRU cache**: prevents redundant Gemini API calls for identical game inputs
- **Zod schema validation**: all API request bodies validated before processing
- **Multi-stage Docker**: minimal production image (~120 MB)

## 4. Innovation

**Criterion**: Does the project introduce novel ideas or creative combinations?

**Evidence**:
- **Real-time personalised AI video**: No two game sessions produce the same video.
  Veo 3.1 receives a Gemini-authored prompt encoding the user's exact choices, then
  synthesises a unique cinematic clip.
- **Multilingual audio overlay**: Cloud TTS narrates the video in the user's chosen
  language (English / Spanish / Hindi), synchronised to the Veo timeline.
- **Gamified carbon scoring**: Eco-points and Carbon-Saved meters update live as
  choices are made, turning climate data into an engaging feedback loop.
- **Anonymous global leaderboard**: Players can see their global rank without
  creating an account, reducing friction while maintaining engagement.

## 6. Security

**Criterion**: Is user data handled securely?

**Evidence**:
- **Helmet CSP**: restricts executable scripts to self and Google Cloud zones
- **express-rate-limit**: prevents Vertex AI resource abuse (10 req/min per IP)
- **Zod validation**: all API payloads validated and sanitised before use
- **Firebase Auth**: anonymous auth; no PII collected
- **Signed Cloud Storage URLs**: generated video URLs expire after 15 minutes
- **Secret Manager**: API keys injected at runtime, never stored in source

**Files**:
- [`SECURITY.md`](../SECURITY.md)
- [`SECURITY_ARCHITECTURE.md`](../SECURITY_ARCHITECTURE.md)
